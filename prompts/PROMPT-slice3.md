## Стек
- NestJS 11 для новых сервисов; React 19 + Rspack + MF 2.0 для новых remote'ов.
- **Next.js 16 (App Router)** для публичного сайта — ОТДЕЛЬНОЕ приложение, **НЕ** подключать к Module Federation.
- Потребление событий из Срезов 1–2 через `libs/kafka`.

## Что построить

### apps/svc-content (NestJS)
- CRUD сценариев/уроков/программы; шаблоны колод/карточек.
- Сценарии разговора переезжают сюда из svc-ai-dialog (svc-ai-dialog начинает брать их отсюда).

### apps/svc-progress (NestJS) — read-model/аналитика
- Потребляет `learning.review.completed` и события разговора/произношения.
- Агрегаты: streak'и, повторения по дням, число due-карточек, динамика произношения, выученные слова.

### apps/svc-notifications (NestJS)
- Напоминания: дневное напоминание о повторении, «streak под угрозой».
- Планировщик + потребление событий; отправка — заглушка (email/push провайдер).

### apps/mfe-progress (React, MF remote)
- Дашборд: streak'и, статистика, графики (due, повторения во времени, тренд произношения).

### apps/mfe-studio (React, MF remote)
- Админка (роль `admin`): управление сценариями/контентом из svc-content.

### apps/web-public (Next.js 16, App Router)
- Публичный SSR-сайт: лендинг, SEO-мета, индексируемые превью сценариев, заглушка прайсинга.
- Отдельное приложение в Nx-воркспейсе. Никакого MF внутри Next.

## Стек (пинь свежие стабильные патчи; ориентир — июнь 2026)
- Python 3.13/3.14 + FastAPI для AI/Speech сервисов.
- LLM через провайдерский SDK со стримингом; STT (faster-whisper или провайдер); TTS (провайдер/Piper).
- WebSocket для realtime; MinIO (S3-совместимое) для аудио.
- Kafka — через `@confluentinc/kafka-javascript` (TS) и подтверждённый клиент для Python (confluent-kafka).
- React 19 + Rspack + Module Federation 2.0 для нового remote.

## Что построить

### apps/svc-ai-dialog (Python/FastAPI)
- Оркестрация LLM со стримингом ответа.
- **Сценарии** как режимы разговора: собеседование, аэропорт, ресторан, small talk, рабочий созвон (пока system-prompt'ы можно держать в конфиге сервиса; в Срезе 3 переедут в svc-content).
- Состояние диалога (история реплик на сессию).
- Детекция слабых мест: слова/ошибки, на которых пользователь спотыкается → формирует событие `speaking.mistake.detected`.

### apps/svc-speech (Python/FastAPI)
- STT: аудио → текст.
- TTS: текст ответа → аудио (сохранение в MinIO, отдача ссылки/стрима).
- Оценка произношения: начни с провайдерского/эвристического скоринга; пометь forced-alignment как «hard R&D» на будущее, не блокируй им срез.

### apps/mfe-speaking (React, MF remote)
- Выбор сценария; запись микрофона; стрим аудио на бэкенд по WebSocket.
- Живой транскрипт; текст ответа AI (стримом) + воспроизведение TTS; фидбэк по произношению.
- Подключается к host `shell` как remote (MF 2.0, шаринг React/типов).

### Realtime-путь
- WebSocket-эндпоинт (в gateway-bff или отдельной точке) ↔ svc-ai-dialog/svc-speech.
- **Валидация токена Keycloak на WS-handshake** (через libs/auth).
- Аудио-стриминг в обе стороны.

### ⭐ Петля обратной связи (главное в срезе)
- `svc-ai-dialog`/`svc-speech` публикует `speaking.mistake.detected` (user_id, слова/ошибки, контекст).
- `svc-vocabulary` потребляет → добавляет слова в колоду пользователя.
- `svc-learning` потребляет → ставит эти карточки в очередь повторения (`card_schedules.due = now`).
- Результат: слово, проваленное в разговоре, всплывает в очереди повторения Среза 1.

### Контракты между TS и Python (важный нюанс)
- Схемы новых событий — в `libs/contracts`.
- Так как события читают и TS-, и Python-сервисы: определи схемы как **Avro/JSON Schema/Protobuf** и генерируй из них и TS-типы, и Python-модели (или используй Schema Registry). Не дублируй типы вручную в двух языках.

## Definition of Done
- docker-compose дополнен MinIO и Python-сервисами; `nx run-many -t serve` поднимает в т.ч. mfe-speaking, svc-ai-dialog, svc-speech.
